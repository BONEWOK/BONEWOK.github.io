🌱 日常更新：就像写论文一样简单

1. 更新个人信息（比如新发了一篇论文）

假设你刚发表了一篇论文，想在网站上更新。流程和你创建网站时完全一样：

  1. 进入你的项目文件夹my-website，以下是示例

    cd C:\Users\BOOK\Desktop\AAA_linux\site-directed\my-website

  2. 编辑对应的文件

    # 比如在 index.qmd 里更新论文列表，或者在 publications.qmd 里添加新条目

  3. 预览确认（强烈推荐！）

    quarto preview

  4. 重新构建网站

    quarto render

  5. 进入 _site 文件夹并推送到 GitHub

    1. 1 进入 _site 文件夹
    cd _site
    1. 2 初始化 git，并强制推送到你的仓库
    git init
    1. 3 add
    git add .
    1. 4 commit
    git commit -m "deploy website"
    1. 1 remote
    git remote add origin https://github.com/BONEWOK/BONEWOK.github.io.git
    1. 6 branch
    git branch -M main
    1. 7 push
    git push -f origin main

小技巧：如果你只是改文字内容，可以跳过 quarto preview 直接 quarto render，但预览能帮你发现格式问题，避免推到网上才发现不对。

2. 添加新页面（比如增加“教学”或“博客”板块）

  1. 在项目文件夹里新建一个 .qmd 文件，比如 teaching.qmd。

  2. 文件开头写上 YAML 头信息：

    ---
title: "教学"
---

  3. 用 Markdown 写内容。

  4. 在 _quarto.yml 的 navbar 部分添加这个页面的链接：

    navbar:
  left:
    - text: "主页"
      href: index.qmd
    - text: "研究"
      href: research.qmd
    - text: "教学"    # 新加的
      href: teaching.qmd

  5. 然后重复上面的构建和推送步骤。

3.github操作指南

一个完整的 GitHub 项目更改流程通常包括以下步骤：

1. 克隆仓库到本地（如果你还没有本地副本）

如果你没有将 GitHub 仓库克隆到本地，可以使用以下命令：

git clone https://github.com/用户名/仓库名.git cd 仓库名

替换 用户名 和 仓库名 为实际的用户名和仓库名。

2. 创建并切换到一个新分支（可选）

为了保持工作区的整洁，建议在开始进行更改之前创建一个新的分支。这有助于在不影响主分支（main 或 master）的情况下进行更改。

git checkout -b new-branch-name

这将创建并切换到一个新的分支，之后你可以在这个分支上进行更改。

3. 修改文件

进行你需要的更改，例如修改代码、添加新文件或删除不需要的文件。可以使用你喜欢的编辑器进行更改。

4. 查看当前更改

在提交更改之前，查看你所做的更改：

git status

这将显示哪些文件被修改、哪些文件被删除或哪些文件是新添加的。

5. 将更改添加到暂存区

如果你对文件进行了更改，需要将这些文件添加到 Git 的暂存区，以便它们能够被提交。

git add .

上面的命令会将所有更改的文件添加到暂存区。如果你只想添加某个文件，可以指定文件名：

git add <file_name>

6. 提交更改

提交更改时要附上有意义的提交信息：

git commit -m "描述你所做的更改"

提交信息应简洁明了，便于他人理解这次更改的目的。

7. 推送更改到远程仓库

当你提交完更改之后，将本地的更改推送到 GitHub 仓库：

git push origin new-branch-name

如果你是直接在 main 分支上工作，那么命令会是：

git push origin main

8. 创建拉取请求（Pull Request）

在 GitHub 上，推送完分支后，你可以创建一个 Pull Request 来请求将你的更改合并到主分支。

  • 登录到 GitHub，进入你修改的仓库页面。
  • 点击 Compare & pull request 按钮。
  • 填写拉取请求的标题和描述，并选择合适的目标分支（通常是 main 或 master）。
  • 提交 Pull Request，等待项目维护者进行审查和合并。

9. 合并更改

如果你有权限，也可以自己合并拉取请求：

  • 在 GitHub 上，点击 Merge pull request，然后选择 Confirm merge。
  • 如果你没有合并权限，项目管理员将审查你的代码并决定是否合并。

10. 同步更新

如果有人在你的分支之外对 main 分支做了更改，或者你需要确保你的本地仓库和远程仓库是同步的，最好先拉取最新的更改：

git pull origin main

11. 删除本地分支（如果需要）

如果你已经合并了分支并且不再需要该分支，可以删除本地分支：

git branch -d new-branch-name

12. 删除远程分支（如果需要）

如果你不再需要远程分支，可以删除远程分支：

git push origin --delete new-branch-name

总结

  1. 克隆仓库（如果还没克隆）。
  2. 创建新分支（可选）。
  3. 修改文件。
  4. 添加更改到暂存区。
  5. 提交更改。
  6. 推送更改到远程仓库。
  7. 创建拉取请求。
  8. 合并更改。
  9. 同步更新。
  10. 删除本地/远程分支。

如果你在任何步骤中遇到问题，随时告诉我，我可以帮助你解决！

🔄 版本控制：你的“后悔药”

你现在已经在用 Git 了，这是一个强大的时间机器。几个常用命令让你随时可以反悔：

场景 命令 说明
想看看改了什么 git status 显示哪些文件被修改了
想撤销某个文件的修改 git checkout -- 文件名 回到上一次提交的状态
想回到过去某个版本 git log 查看历史，然后 git reset --hard 版本号 谨慎使用，会丢失之后的所有修改
不小心删了文件 git checkout -- 文件名 只要提交过就能恢复

建议：每次更新网站前，先 git pull 拉取最新状态（如果你在多台电脑上维护），避免冲突。

🚨 遇到问题怎么办？

情况1：网站打不开了，显示 404

  • 先去 GitHub 仓库看看，_site 里的文件是否成功推送？index.html 在不在？
  • 检查 Settings → Pages，看看部署状态是不是绿色勾勾。
  • 如果是红色叉叉，点进去看 Actions 的日志，错误信息一般很明确。

情况2：样式乱了，或者公式不显示

  • 大概率是路径问题。检查 _quarto.yml 里有没有设置 output-dir，以及 styles.css 的引用路径是否正确。
  • 在浏览器按 F12 打开开发者工具，看 Console（控制台）里有没有红色报错，通常会告诉你哪个文件没找到。

情况3：想换主题

Quarto 支持很多漂亮的主题，在 _quarto.yml 里修改：

format:
  html:
    theme: cosmo  # 可选：cerulean, journal, flatly, darkly 等

官方主题列表 可以预览效果。

📈 让网站更“学术”的进阶技巧

等你熟悉基础维护后，可以考虑这些功能，对博士生特别有用：

  1. 自动生成论文列表
    如果你有 BibTeX 文件（比如从 Google Scholar 导出的），可以用 Quarto 的 citations 功能自动格式化参考文献列表。

  2. 嵌入动态图表
    如果你有生物信息学数据，可以直接在 .qmd 里写 R 或 Python 代码，Quarto 会在渲染时运行并生成图表。这样你更新数据时，图表会自动更新。

  3. 添加访客统计
    在 _quarto.yml 的 include-in-header 里添加一段 Google Analytics 代码，就能看到谁在访问你的主页。

  4. 绑定自定义域名
    等你以后有了自己的名字（比如 zhangsan.com），可以在 GitHub Pages 设置里绑定，让你的网站看起来更专业。
