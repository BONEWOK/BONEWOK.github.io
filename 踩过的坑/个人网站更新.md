1. 更新个人信息（比如新发了一篇论文）

假设你刚发表了一篇论文，想在网站上更新。流程和你创建网站时完全一样：

1. 进入你的项目文件夹my-website，以下是示例
   
   ```powershell
   cd C:\Users\BOOK\Desktop\AAA_linux\site-directed\my-website
   ```

2. 编辑对应的文件
   
   ```powershell
   # 比如在 index.qmd 里更新论文列表，或者在 publications.qmd 里添加新条目
   ```

3. 预览确认（强烈推荐！）
   
   ```powershell
   quarto preview
   ```

4. 重新构建网站
   
   ```powershell
   quarto render
   ```

5. 进入 _site 文件夹并推送到 GitHub
   
   5. 1 进入 _site 文件夹

```powershell
cd _site
```

5. 2 初始化 git，并强制推送到你的仓库

```powershell
git init
```

5. 3 add

```powershell
git add .
```

5. 4 commit

```powershell
git commit -m "deploy website"
```

5. 5 remote

```powershell
git remote add origin https://github.com/BONEWOK/BONEWOK.github.io.git
```

5. 6 branch

```powershell
git branch -M main
```

5. 7 push

```powershell
git push -f origin main
```

**小技巧**：如果你只是改文字内容，可以跳过 `quarto preview` 直接 `quarto render`，但预览能帮你发现格式问题，避免推到网上才发现不对。
